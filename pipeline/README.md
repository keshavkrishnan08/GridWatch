# Data pipeline

Reproducible scripts that build the static dataset GridWatch renders. No API keys, no database. Run them and the files in `../public/data/` rebuild from their public sources.

## Scripts

| Script | What it does |
|--------|--------------|
| `fetch_geo.py` | Fetches county boundaries, power plants, transmission lines, and utility territories; simplifies the geometry; writes GeoJSON. Degrades gracefully — a dead source becomes an empty layer, never a crash. |
| `build_dataset.py` | Reads `facilities.json` + the geodata and computes statewide roll-ups (load by status, share of state peak, generation mix, per-county and per-utility summaries) into `meta.json`. Pure derivation. |
| `validate.py` | Schema + sanity checks on every file. Add `--links` to HTTP-check every source URL. Exits non-zero on any hard error — CI-ready. |

## Run

```bash
python3 pipeline/fetch_geo.py        # ~30s, pulls the geospatial base layers
python3 pipeline/build_dataset.py    # recompute meta.json
python3 pipeline/validate.py --links # verify schema + that sources resolve
```

Only the Python standard library is required.

## Sources

- **Counties** — US Census cartographic boundaries (plotly mirror), filtered to Indiana (FIPS 18).
- **Power plants** — WRI Global Power Plant Database v1.3.0, clipped to the Indiana bounding box.
- **Transmission** — HIFLD Electric Power Transmission Lines, ≥138 kV, intersecting Indiana.
- **Utility territories** — HIFLD Electric Retail Service Territories, `STATE='IN'`.

The ArcGIS layers are paged (`resultOffset`), simplified server-side (`maxAllowableOffset`), and rounded client-side to keep the static files small and CDN-friendly.

## Curated by hand

`facilities.json` is **not** generated by these scripts. It's curated from IURC filings, utility filings, county records, and reporting, with a source link on every record. That human curation is the credibility core — see [`../METHODOLOGY.md`](../METHODOLOGY.md).

## Fork for another state

1. Change `IN_BBOX` and the `18` FIPS prefix in `fetch_geo.py`.
2. Replace `facilities.json` with your state's projects (same schema, keep `sources`).
3. Re-run all three scripts.
